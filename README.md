# hktuner

A chromatic tuner LV2 plugin for [Haiku OS](https://www.haiku-os.org/), with a native
Interface Kit GUI. MIT licensed.

- **Stereo pass-through**: drop it anywhere in a signal chain; the audio is copied through
  untouched while pitch is detected on the L+R mix.
- **Clean-room pitch detection**: time-domain MPM/NSDF (McLeod & Wyvill, *A Smarter Way to
  Find Pitch*, ICMC 2005), run off the audio thread via the LV2 worker extension. No
  external DSP dependencies.
- **Native Haiku GUI**: an LED-matrix tuner screen (note name, octave, cents strip,
  flat/sharp LED bars) rendered from pre-generated PNG sprites, using the
  [LV2-haiku](https://github.com/rations/LV2-haiku) `HaikuUI` UI type (the widget is a
  `BView*`).
- **Settable reference pitch**: A4 defaults to 440 Hz and can be typed to anything in
  380–500 Hz (432, 415, ...), via the GUI or the `ref_freq` control port.

## Ports

| # | Symbol     | Type        | Direction | Notes                                   |
|---|------------|-------------|-----------|-----------------------------------------|
| 0 | `ref_freq` | Control     | in        | A4 reference, Hz. Default 440, 380–500. |
| 1 | `freq`     | Control     | out       | Detected pitch, Hz. 0 = no pitch.       |
| 2 | `clarity`  | Control     | out       | NSDF peak height, 0–1.                  |
| 3 | `in_l`     | Audio       | in        |                                         |
| 4 | `in_r`     | Audio       | in        |                                         |
| 5 | `out_l`    | Audio       | out       | Bit-exact copy of `in_l`.               |
| 6 | `out_r`    | Audio       | out       | Bit-exact copy of `in_r`.               |

The LV2 worker feature (`work:schedule`) is optional: in a host that doesn't provide it the
plugin still passes audio and reports `freq = 0`.

## Building

Requires a built [LV2-haiku](https://github.com/rations/LV2-haiku) checkout next door (its
`build/stage` provides the LV2 headers via pkg-config) and, on Haiku, gcc and make
(`pkgman install -y gcc make` if missing).

```sh
make                 # builds build/hktuner.lv2 (DSP; plus the BView UI on Haiku)
make install         # copies the bundle to ~/config/non-packaged/add-ons/media/LV2/
make validate        # lv2ls / lv2info against the LV2-haiku stage
LV2_HAIKU=/path/to/LV2-haiku make   # if the checkout is not at ~/LV2-haiku
```

The LV2 spec bundles and the `ui-haiku.lv2` type bundle must be installed once from the
LV2-haiku checkout (`make install-specs` there) so hosts can resolve the plugin metadata.

GUI artwork under `gui/*.png` is generated by the one-shot ImageMagick scripts alongside it
(`gui/make_*.sh`); the outputs are committed, so ImageMagick is not needed to build.
